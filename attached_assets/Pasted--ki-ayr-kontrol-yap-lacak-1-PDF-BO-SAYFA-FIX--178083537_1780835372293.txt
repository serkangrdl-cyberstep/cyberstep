── 1. PDF BOŞ SAYFA FIX ─────────────────────────────────────────────

artifacts/api-server/src/services/pdf.ts dosyasında generateDomainScanPDF fonksiyonunda PDF boş sayfa sorunu var. Sorun: footer döngüsünde switchToPage kullanılınca pageAdded event tetikleniyor ve boş sayfalar oluşuyor.

Şu değişiklikleri uygula:

a) doc.on('pageAdded') listener'ı eklemeden önce bir flag tanımla:
   let headerListenerActive = false;

b) Mevcut doc.on('pageAdded', ...) bloğunu şu şekilde değiştir:
   doc.on('pageAdded', () => {
     if (!headerListenerActive) return;
     _drawHeader();
     doc.y = 52;
   });

c) Kapak sayfası bittikten sonra, ilk içerik sayfası addPage() çağrısından HEMEN ÖNCE şunu ekle:
   headerListenerActive = true;

d) Footer döngüsünden HEMEN ÖNCE şunu ekle:
   headerListenerActive = false;

e) Değişikliği build et ve test et:
   pnpm --filter @workspace/api-server run build
   curl -s -o /tmp/test.pdf -w "%{http_code}" https://cyberstepio-cyber-risk-analyzer.replit.app/api/domain-scan/30/pdf
   pdfinfo /tmp/test.pdf | grep Pages
   
   Beklenen: Pages: 3

── 3. BACKEND STATS ENDPOINTİ KONTROLÜ ──────────────────────────────

artifacts/api-server/src/routes/admin-panel/lead-discovery.ts dosyasında GET /stats endpoint'inin döndürdüğü JSON'da şu alanlar var mı:
- total
- qualified  
- withContact
- teaserSent

Yoksa şu alanları hesaplayıp döndürecek şekilde güncelle:
   const [stats] = await db.execute(sql`
     SELECT
       COUNT(*) as total,
       COUNT(*) FILTER (WHERE is_qualified = true) as qualified,
       COUNT(*) FILTER (WHERE is_qualified = true AND contact_email IS NOT NULL AND contact_email != '') as "withContact",
       COUNT(*) FILTER (WHERE teaser_sent_at IS NOT NULL) as "teaserSent"
     FROM lead_candidates
   `);
