## PART 2 — RISK SCORE CALCULATOR

src/services/executive/riskScoreCalculator.ts oluştur:

export async function calculateCustomerRiskScore(
  customerId: number,
  scanData: CustomerScanSummary
): Promise<number>

Skor hesaplama (0-100, 100 = en iyi):

Başlangıç: 100

Eksiltmeler:
- Her critical CVE: -8 (max -40)
- Her high CVE: -3 (max -15)
- Blacklisted domain: -10 (max -20)
- SSL geçersiz: -8
- SSL 7 günden az: -5
- Mail reputation < 50: -3 (max -9)
- Şüpheli brand variant: -5 (max -15)
- High risk port açık (22,3389,23,5900): -4 (max -12)

Minimum: 0
Skor eşikleri:
- 80-100: Düşük Risk (yeşil)
- 60-79: Orta Risk (amber)
- 40-59: Yüksek Risk (turuncu)
- 0-39: Kritik Risk (kırmızı)

## PART 3 — REPORT DATA COLLECTOR

src/services/executive/reportDataCollector.ts oluştur:

export async function collectCustomerReportData(
  customerId: number,
  periodStart: Date,
  periodEnd: Date
): Promise<ExecutiveReportData>

Toplanan veriler:
1. customers tablosundan müşteri domain'i al
2. domain_scans JOIN lead_candidates WHERE customer domain:
   - critical_cve_count, high_cve_count toplamları
   - blacklist_score < 100 olan sayısı
   - ssl_days_remaining <= 30 olan sayısı
   - mail_reputation_score < 50 olan sayısı
   - open_ports_count toplamı
3. brand_monitors WHERE customer_id:
   - is_active = true sayısı
   - is_suspicious = true sayısı
4. Önceki ayın executive_reports kaydından
   risk_score_current → bu ayın risk_score_previous olur
5. Tüm veriyi riskScoreCalculator'a geç, skor hesapla

Return tipi ExecutiveReportData interface olarak tanımla —
tüm alanlar nullable, hata durumunda kısmi veri dönebilmeli.

## PART 5 — REPORT ORCHESTRATOR

src/services/executive/executiveReportOrchestrator.ts oluştur:

export async function generateMonthlyReports():
  Promise<{ generated: number; errors: number }>

Logic:
1. customers WHERE status IN ('active', 'trial') çek
2. Her müşteri için:
   a. Bu ay raporu zaten var mı? (report_month kontrolü)
      → Varsa atla
   b. collectCustomerReportData() çağır
   c. executive_reports tablosuna kaydet
   d. executivePdfGenerator() ile PDF üret
   e. PDF'i /reports/executive/[customer_id]/[month].pdf kaydet
   f. pdf_path güncelle
3. {generated, errors} döndür

Hata toleransı: bir müşteri hata verirse diğerlerine devam et.

## PART 6 — CRON KAYDI

src/index.ts — mevcut wrapCron pattern:

// Her ayın 1'i 05:00 Istanbul — aylık executive rapor
cron.schedule("0 5 1 * *", wrapCron("executive_report_monthly",
  "0 5 1 * *", async () => {
    const { generateMonthlyReports } = await import(
      "./services/executive/executiveReportOrchestrator"
    );
    return await generateMonthlyReports();
}), { timezone: "Europe/Istanbul" });

EXPECTED_JOBS: { name: "executive_report_monthly", intervalMinutes: 43200 }
CATCH_UP_CRONS'a ekle.

## PART 7 — ADMIN API

src/routes/admin-panel/executive-reports.ts oluştur:

GET /api/admin-panel/executive-reports/list
→ executive_reports JOIN customers
→ ORDER BY generated_at DESC
→ LIMIT 50
→ Her kayıt: { customer_name, customer_domain, report_month,
  risk_score_current, risk_score_change, generated_at, pdf_path }

GET /api/admin-panel/executive-reports/:customerId/history
→ Son 6 ay raporu (risk skoru trendi için)
→ ORDER BY report_month ASC

POST /api/admin-panel/executive-reports/generate-now
→ Body: { customer_id?: number }
  (belirtilmezse tüm aktif müşteriler)
→ generateMonthlyReports() tetikle
→ Response: { message: "started" }

GET /api/admin-panel/executive-reports/:id/download
→ pdf_path'ten dosyayı serve et
→ Content-Type: application/pdf

admin-panel/index.ts'e register et.
