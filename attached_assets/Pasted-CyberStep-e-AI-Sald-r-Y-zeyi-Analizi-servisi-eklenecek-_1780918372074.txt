src/services/attackSurface/ klasörü oluştur. 5 modül:

── 2.1 subdomainDiscovery.ts ─────────────────────

export async function discoverSubdomains(domain: string): Promise<string[]>

Şu kaynakları paralel sorgula (Promise.all):
1. crt.sh: GET https://crt.sh/?q=%.{domain}&output=json
   → unique subdomain listesi çıkar, wildcard'ları filtrele
2. DNS brute force: yaygın prefix listesi dene (www, mail, ftp, api, admin, app, portal, vpn, remote, webmail, smtp, pop, imap, ns1, ns2, dev, staging, test)
   → dns.resolve4 ile her birini çözümle, çözümlenenler listeye gir
3. VirusTotal subdomain API (key varsa):
   GET https://www.virustotal.com/api/v3/domains/{domain}/subdomains

Max 50 subdomain döndür, unique ve doğrulanmış (DNS'te çözümlenen).
Timeout: tüm işlem max 30 saniye.

── 2.2 serviceFingerprint.ts ────────────────────

export async function fingerprintServices(domain: string, subdomains: string[]): Promise<ServiceInfo[]>

Her subdomain + ana domain için:
1. HTTP/HTTPS HEAD isteği at (timeout: 5sn)
2. Response headers'dan teknoloji tespit et:
   - Server header → nginx, Apache, IIS, Caddy
   - X-Powered-By → PHP, ASP.NET, Express
   - Set-Cookie patterns → WordPress, Laravel, Django
   - Via header → CDN tespiti (Cloudflare, Fastly, Akamai)
3. Shodan API'den IP bilgisi çek (SHODAN_API_KEY varsa):
   → açık portlar, servis banner'ları, işletim sistemi

Döndür: Array<{
  host: string,
  ip: string | null,
  technologies: string[],
  openPorts: number[],
  server: string | null,
  cdn: string | null,
  responseCode: number | null
}>

── 2.3 sslAnalyzer.ts ───────────────────────────

export async function analyzeSsl(domain: string): Promise<SslAnalysis>

tls.connect ile bağlan, şunları analiz et:
1. Sertifika: issuer, validFrom, validTo, daysRemaining, SANs
2. Protokol versiyonu: TLS 1.0/1.1 hâlâ aktif mi? (güvenlik riski)
3. Self-signed mı?
4. Chain complete mi?
5. Wildcard sertifika mı?

Döndür: {
  valid: boolean,
  issuer: string,
  daysRemaining: number,
  protocol: string,
  weakProtocols: string[],  // TLS 1.0, TLS 1.1 varsa
  isSelfSigned: boolean,
  isWildcard: boolean,
  sans: string[],
  riskLevel: "low" | "medium" | "high"
}

── 2.4 cveMapper.ts ─────────────────────────────

export async function mapCves(technologies: string[]): Promise<CveMatch[]>

Tespit edilen teknoloji listesini al, her biri için:
1. Versiyon varsa: NVD API sorgula
   GET https://services.nvd.nist.gov/rest/json/cves/2.0?keywordSearch={tech}&resultsPerPage=5
2. Versiyon yoksa: bilinen yüksek riskli CVE'leri eşleştir (local mapping)
3. CVSS skoru 7.0+ olanları "critical" olarak işaretle

Döndür: Array<{
  technology: string,
  cveId: string,
  cvssScore: number,
  severity: "critical" | "high" | "medium",
  description: string,
  publishedDate: string
}>

Max 10 CVE döndür, CVSS skoru yüksekten düşüğe sıralı.
