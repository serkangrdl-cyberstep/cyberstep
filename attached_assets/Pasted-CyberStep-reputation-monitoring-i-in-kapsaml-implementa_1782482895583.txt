## PART 3 — ORCHESTRATOR

src/services/reputationOrchestrator.ts oluştur:

export async function runReputationMonitoring(): 
  Promise<{processed: number, errors: number}>

Logic:
1. domain_scans WHERE blacklist_checked_at IS NULL 
   OR blacklist_checked_at < NOW() - INTERVAL '7 days'
   LIMIT 200 per run
2. Her scan için sırayla:
   runBlacklistCheck() → runSSLCheck() → runMailReputationCheck()
3. Domainler arası 500ms delay (rate limit koruması)
4. Her 50 domain'de log
5. {processed, errors} döndür

## PART 5 — ADMIN API ENDPOINTS

Mevcut admin routes yapısına ekle:

GET /api/admin-panel/reputation/summary
Response: {
  total_checked: number,
  blacklisted_count: number,        // blacklist_score < 100
  blacklist_critical_count: number, // blacklist_score < 50
  ssl_expiring_soon: number,        // days_remaining <= 30
  ssl_critical: number,             // days_remaining <= 7
  ssl_invalid: number,              // ssl_is_valid = false
  poor_mail_reputation: number,     // score < 50
  unchecked_count: number,          // blacklist_checked_at IS NULL
  last_run_at: timestamp | null
}

POST /api/admin-panel/reputation/run-now
→ runReputationMonitoring() tetikle (async, beklemeden)
→ Response: { message: "started", timestamp: ISO string }

GET /api/admin-panel/reputation/alerts
→ domain_scans JOIN lead_candidates WHERE:
  blacklist_score < 100 
  OR ssl_days_remaining <= 30 
  OR ssl_is_valid = false 
  OR mail_reputation_score < 50
→ ORDER BY (kritiklik skoru) DESC
→ LIMIT 50
→ Her kayıt: { domain, blacklist_score, ssl_days_remaining, 
  ssl_is_valid, ssl_issuer, mail_reputation_score, 
  mail_spf_valid, mail_dmarc_valid, blacklist_hits }

## PART 6 — OVERALL SCORE ENTEGRASYONU

Domain risk skoru hesaplanan yeri bul (overall_score).
Şu faktörleri ekle (minimum 0, maksimum değişmez):

if blacklist_score < 50:     overall_score -= 15
if ssl_is_valid = false:     overall_score -= 20
if ssl_days_remaining <= 7:  overall_score -= 15
if mail_reputation_score < 50: overall_score -= 5

overall_score = Math.max(0, overall_score)

## PART 7 — ADMIN PANEL YENİ SAYFA

Mevcut admin panel layout/style kullan.

### Üst Kısım — 4 Özet Kart
Brand renkler: bg #060D1A, cyan #00C8FF, amber #F5A623

Kart 1 — Blacklist:
- Temiz domain sayısı (blacklist_score = 100) → yeşil
- Kara listede (blacklist_score < 50) → kırmızı badge

Kart 2 — SSL:
- Kritik (ssl_days_remaining <= 7 OR ssl_is_valid=false) → kırmızı
- Yaklaşan (8-30 gün) → amber
- Geçersiz sertifika sayısı

Kart 3 — Mail Reputation:
- İyi (score >= 75) → yeşil
- Zayıf (score < 50) → amber
- SPF eksik sayısı

Kart 4 — Genel:
- Toplam kontrol edilen
- Hiç kontrol edilmemiş (IS NULL)
- Acil dikkat gerektiren → kırmızı

Data source: GET /api/admin-panel/reputation/summary

### Alt Kısım — Alerts Tablosu

Kolonlar: Domain | Blacklist | SSL | Mail Rep. | Son Kontrol | Detay

Filtre butonları: Tümü | Sorunlu | Blacklisted | SSL Kritik | Mail Zayıf

Badge renkleri:
- Blacklist: score=100 → yeşil "Temiz", <100 → kırmızı "⚠ Listede"
- SSL: >30g → yeşil, 7-30 → amber "Xg kaldı", <7 → kırmızı "KRİTİK",
  invalid → kırmızı "GEÇERSİZ"
- Mail: >=75 → yeşil, 50-74 → amber, <50 → kırmızı

"Manuel Kontrol Çalıştır" butonu:
→ POST /api/admin-panel/reputation/run-now
→ Loading spinner → tamamlanınca toast

Data source: GET /api/admin-panel/reputation/alerts
