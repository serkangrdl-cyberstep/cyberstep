## 2.2 Fortinet Security Fabric

FortiGate/FortiAnalyzer log ingestion + Claude AI korelasyon motoru + isteğe bağlı FortiManager otomatik bloklama.

### Mimari

| Bileşen | Açıklama |
|---|---|
| **Log İngestiyonu** | HTTPS-only; `POST /api/fabric/webhook/:token`, `syslog/:token`, `verify/:token`. FortiAnalyzer'ın HTTPS log-forward endpoint'i kullanır |
| **Parser** | FortiGate syslog, CEF, JSON format desteği. `subtype` alanı `type`'a tercih edilir (UTM logları için) |
| **AI Korelasyon** | Claude Sonnet ile Türkçe korelasyon raporu; kritik/yüksek olaylar anında e-posta alarmı tetikler |
| **FortiManager Otomasyon** | IP bloklama: mevcut üyeleri al → birleştir → set (üzerine yazma değil, ekleme). AES-256-GCM şifreli credentials |
| **Haftalık Özet** | Her Pazartesi `fabric_weekly_report` cron ile otomatik oluşturulur |

### Güvenlik Kuralları

- `ENCRYPTION_KEY` Replit Secret olarak saklanmalı — `.replit` dosyasına (git-tracked) yazılmamalı
- Ingest token'ları URL path'inde bulunur; pino-http log serializer bu segmentleri maskeler
- Kötü/bilinmeyen token'larda public endpoint her zaman 200 döner (Fortinet cihazlarının retry-storm yaratmasını önler)

## 2.3 ServiceNow Entegrasyonu

SOC vakaları ↔ ServiceNow INC çift yönlü senkronizasyon.

| Özellik | Detay |
|---|---|
| **Bağlantı** | Instance URL + username + API token (AES-256 şifreli), müşteri başına 1 config |
| **Öncelik Eşleme** | critical→1, high→2, medium→3, low→4 |
| **Açma Hook'u** | SOC triage'da yeni case açılınca ServiceNow'da otomatik INC oluşturulur |
| **Kapatma Hook'u** | SOC case resolved/closed/false_positive → ServiceNow INC kapatılır |
| **Senkronizasyon Cron** | Her 15 dakikada bir ServiceNow'da elle kapatılan INC'leri CyberStep'e yansıtır |
| **Müşteri Portalı** | `/hesabim/entegrasyonlarim` → ServiceNow bölümü |
| **Admin Paneli** | `/panel/servicenow` → özet istatistikler, config listesi, incident tablosu |

## 2.5 Bildirim & Alarm Entegrasyonları

SOC olaylarını üç farklı kanala iletebilir. Fire-and-forget `setImmediate` mimarisi.

### Generic Webhook

- HMAC-SHA256 imzalı (`X-CyberStep-Signature`)
- Müşteri başına çoklu webhook endpoint desteği
- SSRF koruması — `validateWebhookUrl()` ile dahili IP'ler engellenir
- Başarısız gönderimler `webhook_deliveries` tablosunda loglanır; `webhook_retry` cron her 10 dakikada yeniden dener
- **Ücretsiz** — credentials DB'de saklanır, env var gerekmez

### Telegram Bot

- MarkdownV2 formatında mesaj
- Müşteri başına 1 bot config (upsert pattern)
- Token geçerliliği kayıt sırasında `getTelegramBotInfo()` ile doğrulanır
- **Ücretsiz**

### NetGSM SMS

- XML body → `https://api.netgsm.com.tr/sms/send/xml`
- Müşteri başına 1 config, birden fazla alıcı telefon numarası
- Response "00" = başarılı
- **Ücretli / kullanım bazlı**

### SOC Tetikleyici Noktaları

| Olay | Tetikleyici |
|---|---|
| `soc.case.opened` | Yeni SOC vakası açılınca |
| `soc.case.critical` | Kritik seviyede vaka açılınca |
| `soc.case.closed` | Vaka kapatılınca |
| `soc.sla.breached` | SLA süresi aşılınca |
